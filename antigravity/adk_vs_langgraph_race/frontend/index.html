Tool & State Definition

from langgraph.graph import StateGraph, START, END from langchain_core.tools import tool @tool def calculate(expression: str) -> str: """Evaluate a math expression.""" return str(eval(expression)) class GraphState(TypedDict): query: str analysis: str reasoning: str final_answer: str steps: list[StepResult]

Graph Construction

# Initialize LLM with tools llm = ChatVertexAI(model="gemini-2.0-flash-001") llm_with_tools = llm.bind_tools([calculate]) # Define node function async def reason_node(state: GraphState) -> GraphState: response = await llm_with_tools.ainvoke(messages) if response.tool_calls: # Execute tools manually result = calculate.invoke(response.tool_calls[0]['args']) # Follow-up call with result final = await llm.ainvoke(follow_up_msgs) return state # Build graph builder = StateGraph(GraphState) builder.add_node("reasoner", reason_node) builder.add_edge(START, "analyzer") builder.add_edge("analyzer", "reasoner") builder.add_edge("reasoner", END) graph = builder.compile()

Key Differences

Aspect Google ADK 2.0 LangGraph
Tool Execution Automatic (framework handles) Manual (developer controls)
State Management Session-based (InMemorySessionService) TypedDict passed through graph
Orchestration Agent per step, separate runners Single compiled graph
LLM Calls (with tools) Auto multi-turn (2+ calls) Explicit calls (developer decides)
Overhead Higher (more abstraction) Lower (direct control)
