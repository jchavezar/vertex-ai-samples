## 🔍 API Deep Dive: Method & Latency Comparison

### Performance Benchmarks (measured on `agentspace-testing`)

| Method | Simple Query (Lat.) | Complex Query (Lat.) | TTFB (Perceived) |
| :--- | :--- | :--- | :--- |
| **Search** | ~2.7s | ~2.5s | Immediate |
| **Answer** | ~8.3s | ~9.2s | ~8s |
| **StreamAnswer** | ~8.2s | ~9.1s | **~3.5s** |
| **StreamAssist** | ~33.2s | ~23.6s | **~3.8s** |

### Comparison Analysis

1. **`:search` (The Speedster)**
    *   **Purpose:** Pure information retrieval.
    *   **Pros:** Lowest latency (~2.6s). Best for building custom search interfaces.
    *   **Cons:** No natural language answer.

2. **`:answer` (The Scholar)**
    *   **Purpose:** Single-shot grounded Q&A.
    *   **Pros:** High trust, comprehensive answer synthesis.
    *   **Cons:** Users wait ~9s for the full response.

3. **`:streamAnswer` (The Balanced Choice)**
    *   **Purpose:** Streaming grounded Q&A.
    *   **Pros:** Same quality as `:answer` but users see the first tokens in **~3.5s**. Best for web apps.

4. **`:streamAssist` (The Reasoner)**
    *   **Purpose:** Interactive AI Assistant (Multi-turn).
    *   **Pros:** Deep reasoning, tool calling, and multi-turn context.
    *   **Cons:** Very high total latency for complex tasks as it explores multiple thoughts, but provides immediate feedback via TTFB.

## 🛠️ Programmatic Agent Management
We confirmed that **Agents (Engines)** can be created programmatically via the API.
*   **Discovery:** An Agent in the UI corresponds to an `Engine` in the API.
*   **Constraint:** You cannot create an empty Search Engine; it **must** be linked to at least one `DataStore` at the time of the `POST` request.

## 🚧 Issues & Obstacles
1.  **Sync Latency:** Connectors like Jira might return "I cannot answer" if the initial sync is incomplete or if credentials have expired.
2.  **API Complexity:** The Discovery Engine API uses nested resource paths (Projects > Locations > Collections > Engines > Assistants > Agents), making direct REST calls verbose.
3.  **Authentication:** Requires `https://www.googleapis.com/auth/cloud-platform` scope and specific IAM roles (`Discovery Engine Editor`).
