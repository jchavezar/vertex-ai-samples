Set your variables in the terminal

Paste this block. The values come from the inputs at the top of this page — edit them up there and this block updates instantly.

export PROJECT_ID="my-gcp-project"
export ATLASSIAN_CLIENT_ID="paste-after-step-1"
export ATLASSIAN_CLIENT_SECRET="paste-after-step-1"

# Derived
export PROJECT_NUMBER=$(gcloud projects describe "$PROJECT_ID" --format='value(projectNumber)')
export REGION="us-central1"
export REPO_ROOT="$(pwd)"   # run from atlassian-jira-integration/
If PROJECT_NUMBER ends up empty, your gcloud auth can't see my-gcp-project — fix that before continuing.

Create the Atlassian OAuth app

  1. developer.atlassian.com/console/myapps/ → Create → OAuth 2.0 integration
  2. Name: gemini-jira-agent
  3. Permissions → Jira API: read:jira-work, write:jira-work, read:jira-user
  4. Authorization → Callback URL: https://vertexaisearch.cloud.google.com/oauth-redirect
  5. Settings → Copy Client ID + Secret → paste them into the input bar at the top of this page. The Step 0 block updates automatically.

Deploy the MCP server to Cloud Run

cd "$REPO_ROOT/option-a-custom-mcp-portal/jira_server"

gcloud run deploy jira-mcp-server \
  --source . \
  --project "$PROJECT_ID" \
  --region "$REGION" \
  --allow-unauthenticated \
  --port 8080 --memory 1Gi --cpu 2 --timeout 600 --max-instances 5

export MCP_SERVER_URL=$(gcloud run services describe jira-mcp-server \
  --project "$PROJECT_ID" --region "$REGION" --format='value(status.url)')/sse
echo "MCP URL: $MCP_SERVER_URL"
Multi-tenant by design: the server reads the per-request Authorization: Bearer <token> header and uses that token to call Jira. No secret embedded at deploy time.

Write the agent's .env (one heredoc, all variables substituted)

This heredoc IS the whole config. No script hides what's happening — you can see my-gcp-project lands in 3 places.

cat > "$REPO_ROOT/option-a-custom-mcp-portal/adk_agent/.env" <<EOF
# Google Cloud — same value, three keys; libraries look for different names.
GOOGLE_CLOUD_PROJECT=my-gcp-project
GOOGLE_CLOUD_QUOTA_PROJECT=my-gcp-project
GOOGLE_CLOUD_PROJECT_NUMBER=${PROJECT_NUMBER}
GOOGLE_CLOUD_LOCATION=us-central1
GOOGLE_GENAI_USE_VERTEXAI=True
STAGING_BUCKET=gs://my-gcp-project-agent-staging

MCP_SERVER_URL=${MCP_SERVER_URL}
MCP_SERVICE_RESOURCE=projects/${PROJECT_NUMBER}/locations/us-central1/services/jira-mcp-server

ATLASSIAN_CLIENT_ID=paste-after-step-1
ATLASSIAN_CLIENT_SECRET=paste-after-step-1

AGENTSPACE_AUTH_ID=atlassian-jira-oauth
EOF

chmod 600 "$REPO_ROOT/option-a-custom-mcp-portal/adk_agent/.env"

Deploy the ADK agent to Vertex AI Agent Engine

cd "$REPO_ROOT/option-a-custom-mcp-portal/adk_agent"
pip install -r requirements.txt
python deploy_agent_engine.py

Save the Reasoning Engine ID printed at the end.

Register OAuth + agent in Gemini Enterprise

cd "$REPO_ROOT/option-a-custom-mcp-portal"
python register.py all

Reads adk_agent/.env, no prompts. Creates the GE Authorization, registers the agent, shares with ALL_USERS.

Test

  1. Open the Gemini Enterprise app in the cloud console.
  2. Agent picker → Jira Agent
  3. Try: "List 5 Jira issues created this week."
  4. First time: Atlassian consent popup → log in → pick site → accept.
  5. Answers come back with clickable issue keys.
